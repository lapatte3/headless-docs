This page provides definitions for terms related to headless WordPress.

## A

### API

Application Programming Interface: A method for one application or piece of software to communicate with another.

### Atlas

[Atlas](https://wpengine.com/atlas/) is the complete headless WordPress platform. Atlas includes dynamic Node.js and headless WordPress in one package with your choice of modern front-end framework.

### Atlas Content Modeler

[Atlas Content Modeler](https://wordpress.org/plugins/atlas-content-modeler/) is an open-source WordPress plugin from WP Engine that simplifies the process of creating content models for your headless WordPress site. With a single tool, you can create a new content model, define the custom fields and taxonomies for it, develop relationships between the model and other posts or users, and have all of that data automatically available to you via the WordPress REST API or WPGraphQL.

## B

### Backend

Code that runs server-side (on the web server), as opposed to client-side (in the site visitor's web browser).

### Buildtime

The period during which programming code is converted to the compiled assets ([HTML](#html), [CSS](#css), [JavaScript](#javascript), images, and other files) that will ultimately be served to end users. Buildtime is also commonly referred to as "build time", "compiletime", or "compile time".

## C

### Content Management System (CMS)

A web application that is used to manage content. [WordPress](#wordpress) is the most popular CMS, powering over 40% of the web.

### CSS

A style sheet language used to describe the presentation of a webpage. You can learn more on this MDN Web Docs page: [CSS: Cascading Style Sheets](https://developer.mozilla.org/en-US/docs/Web/CSS).

## D

### Decoupled

Decoupling describes the separation of different concerns. In [headless WordPress](#headless-wordpress) architectures, this most commonly means separating the [frontend](#frontend) from the [backend](#backend). This is in contrast to traditional WordPress architectures where the frontend and backend are tightly coupled and exist in a single, unified codebase.

## E

## F

### Faust.js

[Faust.js](https://faustjs.org/) is an open-source JavaScript framework for [headless WordPress](#headless-wordpress). Faust.js consists of a [WordPress](#wordpress) plugin, a set of [npm](#npm) packages, and guides to get you started building headless WordPress sites.

### Frontend

Code that runs client-side (in the site visitor's web browser), as opposed to server-side (on the web server).

## G

### Gatsby

[Gatsby](https://www.gatsbyjs.com/) is an open-source [JavaScript](#javascript) development framework built on top of [Node.js](#node.js) and [React](https://reactjs.org/) for building web applications.

### GQty

[GQty](https://gqty.dev/) is a proxy-based [GraphQL](#graphql) client.

### GraphQL

An open-source data query and manipulation language. In [headless WordPress](#headless-wordpress) architectures, GraphQL is often used to power the [API](#api) layer that the [frontend](#frontend) and [backend](#backend) use to communicate with one another.

## H

### HTML

The markup language used for defining the structure of web pages. You can learn more on this MDN Web Docs page: [HTML: HyperText Markup Language](https://developer.mozilla.org/en-US/docs/Web/HTML).

### Headless CMS

A [content management system](#content-management-system-cms) where the [backend](#backend) content repository (the "body") is [decoupled](#decoupled) from the [frontend](#frontend) presentation layer (the "head").

### Headless WordPress

A website architecture where the [WordPress](#wordpress) [backend](#backend) is [decoupled](#decoupled) from a [frontend](#frontend) application. The WordPress backend and the frontend app typically communicate with one another via the [WordPress REST API](https://developer.wordpress.org/rest-api/) or [WPGraphQL](#wpgraphql).

## J

### Jamstack

Jamstack (or "JAMstack") refers to a modern web architecture using [JavaScript](#javascript), [APIs](#api), and [HTML](#html). From [JamStack.org](https://jamstack.org/): “It’s a new way of building websites and apps that delivers better performance, higher security, lower cost of scaling, and a better developer experience.”

### JavaScript

JavaScript, often abbreviated "JS", is a programming language that is one of the core technologies of the web, alongside [HTML](#html) and [CSS](#css). It can be used to make web pages interactive for website visitors in various ways. With the advent of [Node.js](#node.js), it is also possible to run JavaScript on the server-side, in addition to client-side (in the web browser).

### JSX

A syntax extension to [JavaScript](#javascript) that makes it easier to write code that ultimately gets compiled to [HTML](#html). JSX is most commonly used in [React](https://reactjs.org/) applications. You can learn more by reading this React docs page: [Introducing JSX](https://reactjs.org/docs/introducing-jsx.html).

## N

### Next.js

An open-source [JavaScript](#javascript) development framework built on top of [Node.js](#node.js) and [React](https://reactjs.org/) for building web applications.

### Node.js

[Node.js](https://nodejs.org/en/) is an open-source [JavaScript](#javascript) [runtime](#runtime) environment, typically used to execute JavaScript code on a web server.

### npm

[npm](https://www.npmjs.com/) is a package manager for the [JavaScript](#javascript) programming language maintained by npm, Inc. Despite the fact that "npm" stands for "Node Package Manager", it is commonly used as a package manager for both client-side (JavaScript executed in a web browser) and server-side ([Node.js](#node.js) apps) JavaScript projects.

### Nuxt.js

An open-source [JavaScript](#javascript) development framework built on top of [Node.js](#node.js) and [Vue](https://vuejs.org/) for building web applications.

## P
### Polylang

A WordPress plugin that enables multilingual capabilities translating posts, pages, media, categories, post tags, custom post types and taxonomies, RSS feeds; RTL scripts are supported.


## Q

### Query

A request for information. In the context of web applications, a "query" often refers to the practice of fetching data from a database, or fetching data from an [API](#api) via an [HTTP](https://en.wikipedia.org/wiki/Hypertext_Transfer_Protocol) request.

## R

### React

[React](https://reactjs.org/) is an open-source [JavaScript](#javascript) development framework for building web applications.

### Runtime

The period during which a computer program is executing.

## S

### Schema

The organization or structure of a database or [API](#api).

### Svelte

[Svelte](https://svelte.dev/) is an open-source [JavaScript](#javascript) compiler. Svelte shifts the work traditionally done in the browser into a compile step that happens when you build your app.

### SvelteKit

An open-source [JavaScript](#javascript) development framework built on top of [Node.js](#node.js) and [Svelte](#svelte) for building web applications.

### Vue.js

[Vue](https://vuejs.org/) is an open-source [frontend](#frontend) framework for building user interfaces and single-page applications.

## W

### WordPress

[WordPress](https://wordpress.org/) is an open-source content management system. WordPress powers more than 40% of the top 1 million websites by traffic.

### WPGraphQL

[WPGraphQL](https://www.wpgraphql.com/) is an open-source [WordPress](#wordpress) plugin that provides an extendable [GraphQL](#graphql) [schema](#schema) and [API](#api) for any WordPress site.
